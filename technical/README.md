# Technical

Core cybersecurity knowledge, organized by domain.

This section covers the technical foundations that underlie everything else in security: how systems work, how they fail, and how defenders think about protecting them.

---

## Domains

| Domain | Description |
|---|---|
| [Networking](networking/) | Protocols, traffic analysis, and network-level attack surfaces |
| [Linux](linux/) | Filesystem, permissions, processes, hardening |
| [Windows](windows/) | Authentication, registry, event logs, hardening |
| [Web Security](web-security/) | Application vulnerabilities, OWASP, HTTP, authentication |
| [Active Directory](active-directory/) | AD architecture, Kerberos, attack paths, hardening |
| [Cloud Security](cloud-security/) | IAM, misconfigurations, containers, shared responsibility |
| [Cryptography](cryptography/) | Encryption, hashing, PKI, TLS, and how crypto fails |
| [Malware Analysis](malware-analysis/) | Static and dynamic analysis, persistence, detection |
| [Incident Response](incident-response/) | IR lifecycle, forensics, log analysis, reporting |
| [Threat Hunting](threat-hunting/) | Hypothesis-driven hunting, MITRE ATT&CK, detection engineering |

---

## How This Section Is Organized

Each domain has its own directory. Inside each directory:

- A `README.md` index listing all available files in that domain
- Individual topic files covering specific concepts, techniques, or tools

Topic files follow the format defined in [_templates/topic.md](../_templates/topic.md).

---

## What "Technical" Means Here

The technical section is not a textbook. Each topic is written to explain:

- What the concept is
- How it works at a level that matters for security
- How attackers exploit it or how defenders use it
- Where to go deeper

The goal is not to replace documentation or certifications. It is to build the kind of understanding that makes reading documentation faster and more meaningful.
